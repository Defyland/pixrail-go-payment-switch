# Non-goals

PixRail intentionally does not own:

- customer balances
- double-entry ledger
- settlement accounting
- refunds, chargebacks, or MED case handling
- banking provider certification
- customer-facing wallet UX
- analytics warehouse implementation
- multi-region payment processing

These exclusions keep PixRail focused on payment-rail decisions, routing, idempotency, SPI lifecycle simulation, and event publication.
